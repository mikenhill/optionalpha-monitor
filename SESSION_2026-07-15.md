### Scanner Configuration Semantics
- A scanner is a routine tree containing **Decision** nodes and one or more **Open** action nodes.
- Do not mix decision criteria into Position Criteria:
  - **Entry Criteria:** all decision-node criteria, presented as numbered decisions.
  - **Position Criteria:** only the Open action's `input.filter` values (e.g. reward/risk, max profit).
  - **Position Details:** Open action trade parameters, such as symbol, position size, expiration, legs, price, exits, and tags.
- The scanner modal retains **Raw Routine** JSON for troubleshooting OptionAlpha shapes not yet mapped to readable fields.

### CSV Export Design and Validation (2026-07-17)
- Bulk export was intentionally removed because it would make a `bots.load`, `triggers.list`, and `routines.details` request across all bots/scanners; this is unnecessarily heavy on OptionAlpha APIs.
- **Export CSV** is available only inside the scanner modal, after that scanner's single `routines.details` response has loaded. It is generated locally in the browser and makes no extra API call.
- Output is a long-format comparison file with columns:
  - Bot/scanner context: `bot_name`, `bot_status`, `bot_account`, `bot_group`, `scanner_name`, `scanner_index`.
  - Comparison fields: `category`, `path`, `name`, `value`.
  - Routine metadata: `routine_version`, `routine_updated`.
- Path convention:
  - Decision criterion: `Scanner/Decision[decision text]`, with one row per criterion.
  - Action filter: `Scanner/Action[action text]/Filter`.
  - Trade input: `Scanner/Action[action text]/field`.
- Validated export example: `Put Spread Scanner` v25 generated separate rows for its two Decision nodes, each individual decision criterion, an action filter (`Max profit 70`), and Position Details including symbol, size, price, expiry, put legs, and exits.
- Known export artifact: some Open-action inputs may include empty object-valued keys such as `entry: {}`. These do not represent decision criteria and should be ignored or removed in a later cleanup if they recur.
