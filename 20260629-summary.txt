# Session Summary - 2026-06-29

## New GEX Architecture

**Database Layer:**
- `gex_strike_window` table stores structured strike data (40-strike window) with primary key `(ndate, ntime, symbol, source)`
- `source` column enables testability: `gex` for production, `test` for testing payloads
- JSON array stores strike metrics: `cg`, `pg`, `abs`, `coi`, `poi`, `cvol`, `pvol`, `strike`, `net`

**Controller Layer:**
- `controllers/gex_calculations.py`: Pure calculation functions for each metric (sentiment, gex_ratio, net_gex, kcs, dominance, key_strike_stats, total_oi_and_vol, total_gex, flip_level)
- `controllers/gex_controller.py`: `GexController` with `get_gex_snapshots()` reading from `gex_strike_window`
- `controllers/snapshot_controller.py`: `upsert_gex()` API handles both OptionAlpha historical and live formats

**API Layer (all under /mvc/api/gex/...):**
- `POST /mvc/api/gex/upsert`: Unified upsert API with source/test params
- `GET /mvc/api/gex/strike-window`: Fetch raw strike window data (JSON)
- `GET /mvc/api/gex/strike-window/csv`: Fetch raw strike window data (CSV)
- `GET /mvc/api/gex/compare`: Compare two records for testing
- `GET /api/gex/snapshots`: Fetch calculated snapshots from `gex_strike_window`

**UI Layer:**
- `/gex` route with `gex.html` template (new tab)
- All templates updated with GEX tab navigation

## DAO Usage Pattern

- Use `dao.database.get_connection()` context manager for all DB operations
- Always call `con.commit()` after INSERT/UPDATE operations
- Controllers should be thin: delegate to calculation functions, DAOs for data access
- Decorator `@with_test_metadata(dao_name="...")` for testability tracking

## Testability Features

- **Source isolation**: `source='test'` for test data, `source='gex'` for production
- **Test param**: `?test=1` forces source='test' regardless of source param
- **Compare API**: Compare test vs production records to validate changes
- **Postman collection**: Both historical and live format test requests included
- **OptionAlpha format support**: API accepts wrapped format `[{ "t": "res", "data": { ... } }]`

## Calculation Fixes

- **net_gex**: `call_gex + put_gex` (put_gex already negative)
- **gex_ratio**: Compare absolute values, sign based on which is larger
- **kcs**: Weighted formula (0.5*gex_share + 0.3*oi_share + 0.2*vol_share) * proximity * 100
- **dominance**: `key_abs / total_abs * 100`
- **key_strike_stats**: Proximity-weighted key strike selection
- **flip_level**: Zero-crossing of cumulative net GEX within 40-strike window
