# GEX Intraday Viewer — System Design

## 1. Purpose

A local Flask web application for analysing SPX Gamma Exposure (GEX) using two distinct data paths:

- **Historical analysis** — intraday snapshot files captured from the OptionAlpha `histgex` API.
- **Live analysis** — current market data from the OptionAlpha `market.gex` API plus automated daily summarisation.

The design keeps the two paths separate at the data source level while reusing the same classification, thesis, and verdict logic.

## 2. High-level Requirements

| ID | Requirement |
|----|-------------|
| R1 | Display SPX 5-minute OHLC price charts alongside GEX strike profiles. |
| R2 | Load and visualise historical GEX snapshots taken at fixed times during the trading day. |
| R3 | Generate an End-of-Day (EOD) analysis from the **10:00** (or 9:30 fallback) historical snapshot and compare it to actual SPX OHLC. |
| R4 | Fetch live GEX data on demand and store it as a JSON snapshot. |
| R5 | Automate the post-fetch pipeline: run the daily summary script and generate a concise markdown report. |
| R6 | Provide a per-date **Archived Live** view so historical and live analyses for the same day can be compared. |
| R7 | Keep historical analysis independent of live/daily-summary artifacts so older dates remain analysable. |
| R8 | Use the same classification, thesis, and verdict logic for both historical and live data. |

## 3. Data Sources

| Data | Source | Location |
|------|--------|----------|
| SPX 5-minute OHLC | CSV files | `spx-5min.csv`, `spx-5min-20250201.csv` |
| Historical GEX snapshots | OptionAlpha `histgex` API | `results/histgex/YYYYMMDD/*.json` |
| Live GEX snapshots | OptionAlpha `market.gex` API | `results/livegex/YYYYMMDD/*.json` |
| Daily concise summary | Generated by `optionalpha_daily-summary.py` | `results/daily_gex_summary-concise.csv` |

## 4. UI Tabs

| Tab | Purpose | Data Used |
|-----|---------|-----------|
| **Historical** | Browse intraday GEX snapshots and SPX price for any historical date. | `histgex` + SPX CSV |
| **Live** | Trigger capture/fetch pipelines and view the current market GEX snapshot. | `market.gex` + daily summary |
| **Analysis** | Two panels per selected date: **Historical EOD Analysis** and **Archived Live**. | `histgex` + SPX CSV + daily summary + `analysis/` files |

## 5. Input Commands / Scripts

### 5.1 Manual trigger from the web UI

| Button | API Endpoint | What it runs |
|--------|--------------|--------------|
| **Capture Session** | `GET /api/live/capture` | `optionalpha_capture.py` (refreshes Playwright session cookies) |
| **Fetch Live GEX** | `GET /api/live/fetch` | `optionalpha_daily.py --symbol SPX` → `optionalpha_daily-summary.py` → `generate_concise_report()` |
| **Load Analysis** | `GET /api/analysis?date=YYYY-MM-DD` + `GET /api/archived-live?date=YYYY-MM-DD` | Generates historical EOD analysis and loads archived live artifacts |

### 5.2 Background scripts

| Script | Purpose | Input | Output |
|--------|---------|-------|--------|
| `optionalpha_capture.py` | Refresh Playwright authentication session. | None | Session cookies / state |
| `optionalpha_daily.py --symbol SPX` | Fetch live GEX, append to daily summary, backfill OHLC. | OptionAlpha API | `results/YYYYMMDD_*_SPX_gex.json`, `results/daily_gex_summary.csv` |
| `optionalpha_daily-summary.py` | Build the concise daily summary CSV and HTML dashboard. | Raw GEX JSON files | `results/daily_gex_summary-concise.csv`, `results/daily_gex_summary.html` |

## 6. Output Files

| File | Generated By | Purpose |
|------|--------------|---------|
| `results/histgex/YYYYMMDD/*.json` | External capture process | Historical intraday GEX snapshots |
| `results/livegex/YYYYMMDD/*.json` | `api_live_fetch` | Live GEX snapshots |
| `results/daily_gex_summary.csv` | `optionalpha_daily.py` | Raw rolling daily summary |
| `results/daily_gex_summary-concise.csv` | `optionalpha_daily-summary.py` | Concise daily summary used by live analysis |
| `results/daily_gex_summary.html` | `optionalpha_daily-summary.py` | HTML dashboard |
| `analysis/analysis-concise-YYYYMMDD-HHMM.md` | `generate_concise_report()` | Markdown report generated after a live fetch |

## 7. Core Logic

### 7.1 Snapshot summarisation

- Reads a GEX snapshot (historical or live).
- Computes net GEX, total call/put OI, total call/put volume, sentiment percentage, and GEX ratio.
- Identifies the largest absolute GEX strike as the **wall**.
- Identifies the zero-crossing strike as the **flip**.

### 7.2 Setup classification

Used for both historical and live data. A single helper, `build_thesis()`, classifies the setup based on:

- Net GEX regime (positive / negative).
- Distance from the current price to the wall (PIN / MAGNET / distant wall).
- GEX balance at the wall vs OI net (CALL WALL / PUT PILLAR / balanced PIN / Key OI cluster).
- Volume divergence (heavy call/put volume contradicting the wall character).

### 7.3 Historical EOD analysis

1. Load the 10:00 historical snapshot (fallback to 9:30).
2. Build the thesis using `build_thesis()`.
3. Compare the thesis to actual SPX OHLC from the SPX CSV files.
4. Return thesis, actuals, and verdict.

### 7.4 Live analysis

1. Load the latest daily summary row for the date.
2. Build the thesis using `build_thesis()` with the same classification rules.
3. Compare the thesis to actual OHLC from the daily summary CSV (falling back to SPX CSV if needed).
4. Return thesis, actuals, and verdict.

### 7.5 Archived live lookup

For any date, the `/api/archived-live` endpoint returns:

- The daily summary row (if present).
- The live analysis generated from that row.
- A list of saved `analysis-concise-*.md` files.
- A list of raw JSON captures for that date.

## 8. API Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /api/dates` | Available historical dates. |
| `GET /api/snapshot?date=YYYY-MM-DD&time=HHMM` | Historical GEX snapshot data. |
| `GET /api/analysis?date=YYYY-MM-DD` | Historical EOD analysis. |
| `GET /api/archived-live?date=YYYY-MM-DD` | Live/daily-capture artifacts for a date. |
| `GET /api/live/snapshots` | List of live snapshots for today or a given date. |
| `GET /api/live/snapshot?date=YYYY-MM-DD&time=HHMM` | Specific live snapshot data. |
| `GET /api/live/capture` | Run `optionalpha_capture.py`. |
| `GET /api/live/fetch` | Run the full live pipeline and return the latest snapshot. |
| `GET /api/live/analysis` | Return current-day live analysis. |

## 9. Environment

- **Runtime**: Python 3.12 with Flask.
- **Virtual environment**: `.venv\Scripts\python.exe` is used when spawning external scripts.
- **Server port**: `5051`.
- **Timezone**: Eastern Time (`America/New_York`) for all market-time calculations.
